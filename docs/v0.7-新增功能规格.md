# Hope v0.8 新增功能规格

> 版本：v0.8.0（历史规格；实现状态以根目录 `Hope-产品与技术方案.md` v0.14 为准）

## 概述

v0.8 新增 5 个功能：

1. **进度条位置与方向设置**：支持顶/底/左/右边，方向可选，"我全都要（四边环绕）"作为高级设置中的附加状态
2. **庆祝模式（四边闪烁）**：与 `keep`/`blink`/`hide` 同级互斥的到期提醒模式
3. **任务级位置覆盖**：允许为单个任务指定展示位置（高级设置中开启）
4. **任务完成按钮**：标记任务完成，重复任务自动进入下个循环
5. **颜色去重优化**：不再限制重复颜色，保存时弹出对话框提示用户确认

## 1. 进度条位置与方向设置

### 1.1 功能分级策略

| 设置项 | 位置 | 说明 |
|--------|------|------|
| **进度条位置** | 全局设置 | 顶边 / 底边 / 左边 / 右边；默认**顶边** |
| **进度条方向** | 全局设置 | 顶边/底边：从左到右 / 从右到左；左边/右边：从上到下 / 从下到上；默认智能设定（见 §1.2） |
| **我全都要（四边环绕）** | 高级设置 | 勾选后，从当前位置出发沿当前方向环绕屏幕四边 |

### 1.2 默认方向智能设定

| 位置 | 默认方向 | 说明 |
|--------|----------|------|
| 顶边 / 底边 | 从左到右（_forward_） | 符合阅读习惯 |
| 左边 / 右边 | 从上到下（_forward_） | 符合垂直阅读习惯 |

用户可在高级设置中手动反转方向（_reverse_）。

### 1.4 渲染层改造

#### 四边环绕模式（"我全都要"）

1. 当高级设置中 `AllFour == true` 时创建 **4 个** `OverlayWindow`，分别吸附到屏幕四边。
2. `AllFour` 与基础 `BarPosition`/`BarDirection` 联动：从基础位置出发，沿基础方向环绕。
3. 环绕方向推导：

| 基础位置 `BarPosition` | 基础方向 `BarDirection` | 环绕方向 | 四边顺序（从起始边开始） |
|------------------------|------------------------|----------|--------------------------|
| top | forward | 顺时针 | top → right → bottom → left |
| top | reverse | 逆时针 | top → left → bottom → right |
| bottom | forward | 逆时针 | bottom → right → top → left |
| bottom | reverse | 顺时针 | bottom → left → top → right |
| right | forward | 顺时针 | right → bottom → left → top |
| right | reverse | 逆时针 | right → top → left → bottom |
| left | forward | 逆时针 | left → bottom → right → top |
| left | reverse | 顺时针 | left → top → right → bottom |

4. 后端按主屏幕工作区物理周长计算进度落点：
   - 总周长 `P = (ScreenWidth + ScreenHeight) * 2`
   - 对每项任务，根据其实时进度 `globalPercent` 计算：`x = P * globalPercent / 100`
5. 以 **从顶部、从左到右（顺时针）** 为例，按 `x` 所处区间决定渲染哪些边：

| 阶段 | `x` 范围 | 已填满的边 | 当前填充的边 | 当前边长度 | 当前边方向 | 图片挂载位置 |
|------|----------|------------|--------------|------------|------------|--------------|
| 1 | `0 ≤ x ≤ W` | 无 | 顶部 | `x` | 左→右 | 进度条下方 |
| 2 | `W ≤ x ≤ W+H` | 顶部 | 右侧 | `x - W` | 上→下 | 进度条左侧 |
| 3 | `W+H ≤ x ≤ 2W+H` | 顶部、右侧 | 底部 | `x - W - H` | 右→左 | 进度条上方 |
| 4 | `2W+H ≤ x ≤ P` | 顶部、右侧、底部 | 左侧 | `x - 2W - H` | 下→上 | 进度条右侧 |

6. 每个窗口只渲染分配到该边的 `Segment`。
7. **图片只跟随当前正在填充的边**；已填满的边不再挂载该任务的图片。
8. **图片旋转**：当前活跃边相对于起始边沿环绕方向前进 `n` 条边，则旋转 `n × 90°`（顺时针取正，逆时针取负），旋转中心为图片的吸附边中点。这样同一张图片在四边环绕时会始终从对应边向屏幕内侧"长出"，图片的"厚度"（高度）始终保持一致。
9. **图片尺寸**：四边环绕中所有边的图片均统一按高度限制（`ImageMaxSize`），不再在垂直边限制宽度，确保视觉上大小一致。

#### 坐标映射（伪代码）

```csharp
// 单条边模式
if (BarDirection == "forward") {
    localPercent = globalPercent;
} else {
    localPercent = 100 - globalPercent;
}

// 四边模式（后端计算，桌面端仅按 Position 与 ImageRotation 展示）
// startPos = BarPosition, baseDir = BarDirection
// derive(startPos, baseDir) => clockwise / counterClockwise
// order = sideSequence(startPos, clockwise | counterclockwise)
// W = ScreenWidth, H = ScreenHeight, P = 2*(W+H)
// x = P * globalPercent / 100
//
// 示例：startPos=top, baseDir=forward => clockwise, order = [top, right, bottom, left]
//   x in [0, W]     => top,    length = x,           direction = left->right,  image below,  rotation=0
//   x in [W, W+H]   => right,  length = x - W,       direction = top->bottom,  image left,   rotation=90
//   x in [W+H, 2W+H]=> bottom, length = x - W - H,   direction = right->left,  image above,  rotation=180
//   x in [2W+H, P]  => left,   length = x - 2W - H,   direction = bottom->top,  image right,  rotation=270
// 其中 order 中位于当前活跃边之前的边生成已填满段（FillEnd=100），不挂载图片；
// 仅当前活跃边挂载图片，并设置 ImageRotation。
```

### 1.5 窗口管理改造

#### `App.xaml.cs` 改造

```csharp
// 现有
private OverlayWindow _overlay;

// 改造后
private List<OverlayWindow> _overlays = new();

private void CreateOverlays(SettingsDto settings)
{
    // 关闭现有窗口
    foreach (var overlay in _overlays)
    {
        overlay.Close();
    }
    _overlays.Clear();

    var positions = new List<string>();
    if (settings.AllFour)
    {
        // 四边环绕：始终创建四边窗口，图片旋转由 Segment.ImageRotation 控制
        positions.AddRange(new[] { "top", "right", "bottom", "left" });
    }
    else
    {
        // 单边模式：创建1个窗口
        positions.Add(settings.BarPosition);
    }

    foreach (var pos in positions)
    {
        var window = new OverlayWindow
        {
            Position = pos,
            Direction = LocalDirectionFor(pos, settings.BarPosition, settings.BarDirection, settings.AllFour)
        };
        _overlays.Add(window);
        window.Show();
    }
}

// 单条边模式：根据该边的位置与基础方向返回 forward/reverse。
// 四边环绕时：后端已把每个 Segment 的方向映射到该边的物理方向，
// 前端按 Position 渲染即可，无需额外方向转换。
private string LocalDirectionFor(string position, string barPosition, string barDirection, bool allFour)
{
    if (!allFour) return barDirection;
    // 四边环绕时，每个边的 Segment 方向由后端计算并编码到 BarStart/BarEnd/FillEnd 中。
    return "forward";
}
```

## 2. 庆祝模式（四边闪烁）

### 2.1 功能定义

**庆祝模式（`celebrate`）：**

- 当任务到期时，触发 **四边同时闪烁** 特效。
- 闪烁效果：四边进度条同时做 alpha 渐变脉冲（类似 `blink`，但是四边同时）。
- 持续到用户打开设置查看后停止（类似 `blink` 的 `AcknowledgeBlink` 机制）。

### 2.2 与现有行为的关系

`expiredBehaviors` 现在包含 **4 种显示模式**（单选互斥）+ **1 种通知**（独立勾选）：

| 显示模式 | 说明 |
|---------|------|
| `keep` | 保持显示（100% 满色段常驻） |
| `blink` | 闪烁提醒（该段柔和 alpha 渐变） |
| `hide` | 自动隐藏（从顶栏移出） |
| `celebrate` | 庆祝模式（四边同时闪烁）← **新增** |
| `notify` | 系统通知（勾选框，可叠加在上述任意模式上） |

### 2.3 数据模型

**Go 端 `task.go`：**

```go
// 无需修改，ExpiredBehaviors 已经是 []string
// 只需在验证逻辑中接受 "celebrate"
```

**默认值：**

全局默认 `expiredBehaviors = ["keep"]`（不包含 `celebrate`）。

### 2.4 实现要点

#### `OverlayWindow` 改造

1. 当 `segments[].expired = true` 且 `behaviors` 包含 `celebrate` 时：
   - 如果当前启用"四边环绕"，则四边同时闪烁。
   - 如果当前是单边模式，则仅该边闪烁（降级处理）。
2. 闪烁动画：使用 `DoubleAnimation`（正弦缓动，淡出/淡入）。
3. 停止条件：用户打开设置（`AcknowledgeBlink`）。

#### `App.xaml.cs` 改造

- 广播中新增 `celebrate` 标记。
- 当收到 `celebrate` 任务时，触发四边闪烁。

## 4. 任务完成按钮

### 4.1 功能定义

**"完成"按钮：**

- 在任务列表每个任务项后面添加"完成"按钮（图标或文字）。
- 点击后：
  1. 如果该任务是**重复任务**（循环任务），则自动修改 `startAt`/`endAt` 到下一个循环周期，任务继续运行。
  2. 如果该任务是**单次任务**，则标记为"已完成"，不再渲染到桌面，但保留在任务列表中（带"已完成"标记）。
  3. 如果用户想彻底删除，仍使用"删除"按钮。

### 4.2 数据模型

#### Go 端 `task.go`

无需新增字段。`Recurrence` 字段已包含循环信息。

**"已完成"标记（可选）：**

如果要在任务列表中显示"已完成"状态，需新增字段：

```go

    // ... 现有字段
    CompletedAt *time.Time `json:"completedAt"` // nil = 未完成；非 nil = 完成时间
}
```

### 4.3 实现要点

#### `task.go` 新增 `Complete()` 方法

```go
func (t *Task) Complete(now time.Time) error {
    if t.Recurrence != nil && t.Recurrence.Mode != "" {
        // 重复任务：计算下一个周期
        nextStart := calculateNextOccurrence(t, now)
        if nextStart.IsZero() {
            return errors.New("no next occurrence")
        }
        // 修改 startAt 和 endAt
        duration := t.EndAt.Sub(t.StartAt)
        t.StartAt = nextStart
        t.EndAt = nextStart.Add(duration)
        t.CompletedAt = nil  // 重置完成标记
    } else {
        // 单次任务：标记为已完成
        t.CompletedAt = &now
    }
    return nil
}

// calculateNextOccurrence 计算下一个发生日
func calculateNextOccurrence(t *Task, now time.Time) time.Time {
    if t.Recurrence == nil {
        return time.Time{}
    }
    
    switch t.Recurrence.Mode {
    case "daily":
        // 每天：下一天
        return time.Date(now.Year(), now.Month(), now.Day()+1, 
            t.StartAt.Hour(), t.StartAt.Minute(), 0, 0, t.StartAt.Location())
            
    case "everyN":
        // 间隔 N 天：当前日期 + N
        interval := t.Recurrence.Interval
        if interval < 2 {
            interval = 2  // 最小 2 天
        }
        return time.Date(now.Year(), now.Month(), now.Day()+interval, 
            t.StartAt.Hour(), t.StartAt.Minute(), 0, 0, t.StartAt.Location())
            
    case "weekly":
        // 按星期：找下一个匹配的星期
        // ... 实现略
        
    default:
        return time.Time{}
    }
}
```

## 5. 颜色去重优化

### 5.1 功能定义

**现有行为（v0.6）：**

- 保存任务时，校验颜色是否与其他任务重复。
- 如果重复，阻止保存，提示用户修改。

**新行为（v0.7）：**

- 保存任务时，校验颜色是否与其他任务重复。
- 如果重复，**允许保存**，但弹出对话框提示用户：
  ```
  当前颜色 #FF5733 已被以下任务使用：
  - 写文档
  - 买菜
  
  是否继续使用这个颜色？
  
  [是，继续使用]  [否，修改颜色]
  ```
- 用户选择"是"则继续保存；选择"否"则取消保存，返回编辑。

### 5.2 实现要点

#### `ConfigWindow.xaml.cs` 改造

```csharp
private void SaveTask()
{
    // ... 现有校验（名称、时间等）
    
    // 颜色去重校验
    var duplicateTasks = _rows.Where(r => r != _editingRow && 
                                           r.Color.Equals(_editingRow.Color, StringComparison.OrdinalIgnoreCase)).ToList();
    
    if (duplicateTasks.Any())
    {
        var result = MessageBox.Show(
            $"当前颜色 {_editingRow.Color} 已被以下任务使用：\n" +
            string.Join("\n", duplicateTasks.Select(t => "- " + t.Name)) +
            "\n\n是否继续使用这个颜色？",
            "颜色重复提醒",
            MessageBoxButton.YesNo,
            MessageBoxImage.Warning);
            
        if (result == MessageBoxResult.No)
        {
            return;  // 取消保存
        }
    }
    
    // ... 继续保存
}
```

### 5.3 配置窗 UI

无需新增 UI 元素，只需修改保存逻辑。

## 6. 施工顺序建议

建议按以下顺序施工：

1. **颜色去重优化**（最简单，仅修改保存逻辑）
2. **任务完成按钮**（中等难度，需修改 Go 和 C#）
3. **进度条位置设置**（较难，需改造渲染层）
4. **庆祝模式**（较难，需改造渲染层和动画）
5. **任务级位置覆盖**（最难，需改造多窗口协调）

## 8. 开放问题

1. **屏幕尺寸未上报时的降级**：若 `ScreenWidth` / `ScreenHeight` 为 0，四边模式可临时回退到每边 25% 的均分方案，或等待桌面端下一次 updateSettings 后再精确计算。
2. **庆祝模式的闪烁频率**：建议 1 秒渐出 + 1 秒渐入，循环直到用户查看。
3. **任务级位置覆盖的优先级**：如果全局是"四边模式"，任务是"顶边"，应该如何渲染？（建议：任务级覆盖优先）
4. **"完成"按钮的图标**：建议使用 ✓ 或 🎯（更有"盼头"的感觉）。
