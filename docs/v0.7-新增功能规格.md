# Hope v0.7 新增功能规格

> 版本：v0.7  
> 日期：2026-06-26  
> 状态：文档已更新，待施工

## 概述

v0.7 新增 5 个功能：

1. **进度条位置与方向设置**：支持顶/底/左/右边，方向可选，四边模式放入高级设置
2. **庆祝模式（四边闪烁）**：与 `keep`/`blink`/`hide` 同级互斥的到期提醒模式
3. **任务级位置覆盖**：允许为单个任务指定展示位置（高级设置中开启）
4. **任务完成按钮**：标记任务完成，重复任务自动进入下个循环
5. **颜色去重优化**：不再限制重复颜色，保存时弹出对话框提示用户确认

## 1. 进度条位置与方向设置

### 1.1 功能分级策略

| 设置项 | 位置 | 说明 |
|--------|------|------|
| **进度条位置** | 全局设置 | 顶边 / 底边 / 左边 / 右边；默认**顶边** |
| **进度条方向** | 高级设置 | _forward_ / _reverse_；默认智能设定（见 §1.2） |
| **四边模式** | 高级设置 | 勾选后启用，生成4个 Overlay 窗口吸附四边 |

### 1.2 默认方向智能设定

| 位置 | 默认方向 | 说明 |
|--------|----------|------|
| 顶边 / 底边 | 从左到右（_forward_） | 符合阅读习惯 |
| 左边 / 右边 | 从上到下（_forward_） | 符合垂直阅读习惯 |

用户可在高级设置中手动反转方向（_reverse_）。

### 1.3 数据模型

#### Go 端 `config.go`

```go
type Settings struct {
    // ... 现有字段
    BarPosition  string `json:"barPosition"`  // "top" | "bottom" | "left" | "right"
    BarDirection string `json:"barDirection"` // "forward" | "reverse"
    BarFourSides bool   `json:"barFourSides"` // 是否启用四边模式
}
```

**默认值：**

```go
var DefaultSettings = Settings{
    // ... 现有默认值
    BarPosition:  "top",
    BarDirection: "forward",
    BarFourSides: false,
}
```

**`mergeSettings` 兼容性处理：**

```go
func mergeSettings(target, source Settings) Settings {
    // ... 现有逻辑
    
    // 新增字段兼容性
    if source.BarPosition != "" {
        target.BarPosition = source.BarPosition
    } else if target.BarPosition == "" {
        target.BarPosition = "top"  // 默认值
    }
    
    if source.BarDirection != "" {
        target.BarDirection = source.BarDirection
    } else if target.BarDirection == "" {
        target.BarDirection = "forward"  // 默认值
    }
    
    // BarFourSides 是 bool，零值为 false，无需特殊处理
    target.BarFourSides = source.BarFourSides
    
    return target
}
```

### 1.4 渲染层改造

#### 单条边模式（顶/底/左/右）

1. 根据 `BarPosition` 创建 **1 个** `OverlayWindow`，吸附到对应屏幕边缘。
2. 根据 `BarDirection` 决定填充方向：
   - _forward_：从左到右（水平边）或从上到下（垂直边）
   - _reverse_：从右到左（水平边）或从下到上（垂直边）
3. 图片定位：
   - 水平边：图片中心对齐进度前沿
   - 垂直边：图片中心对齐进度前沿

#### 四边模式

1. 根据 `BarFourSides = true` 创建 **4 个** `OverlayWindow`，分别吸附到屏幕四边。
2. 全局进度 `0–100%` 映射到四条边：
   - 顶边：0% – 25%
   - 右边：25% – 50%
   - 底边：50% – 75%
   - 左边：75% – 100%
3. 每个窗口只渲染落到自己范围内的段。
4. 图片保持在屏幕内侧（不随边旋转）。

#### 坐标映射（伪代码）

```csharp
// 单条边模式
if (BarDirection == "forward") {
    localPercent = globalPercent;
} else {
    localPercent = 100 - globalPercent;
}

// 四边模式
int edgeIndex = (int)Math.Floor(globalPercent / 25.0);  // 0=顶, 1=右, 2=底, 3=左
double localPercent = (globalPercent % 25) * 4;  // 映射到 0-100
```

### 1.5 窗口管理改造

#### `App.xaml.cs` 改造

```csharp
// 现有
private OverlayWindow _overlay;

// 改造后
private List<OverlayWindow> _overlays = new();

private void CreateOverlays(SettingsDto settings)
{
    // 关闭现有窗口
    foreach (var overlay in _overlays)
    {
        overlay.Close();
    }
    _overlays.Clear();
    
    if (settings.BarFourSides)
    {
        // 四边模式：创建4个窗口
        _overlays.Add(CreateOverlayWindow("top"));
        _overlays.Add(CreateOverlayWindow("right"));
        _overlays.Add(CreateOverlayWindow("bottom"));
        _overlays.Add(CreateOverlayWindow("left"));
    }
    else
    {
        // 单边模式：创建1个窗口
        _overlays.Add(CreateOverlayWindow(settings.BarPosition));
    }
    
    // 显示所有窗口
    foreach (var overlay in _overlays)
    {
        overlay.Show();
    }
}

private OverlayWindow CreateOverlayWindow(string position)
{
    var window = new OverlayWindow
    {
        Position = position,
        Direction = settings.BarDirection
    };
    return window;
}
```

## 2. 庆祝模式（四边闪烁）

### 2.1 功能定义

**庆祝模式（`celebrate`）：**

- 当任务到期时，触发 **四边同时闪烁** 特效。
- 闪烁效果：四边进度条同时做 alpha 渐变脉冲（类似 `blink`，但是四边同时）。
- 持续到用户打开设置查看后停止（类似 `blink` 的 `AcknowledgeBlink` 机制）。

### 2.2 与现有行为的关系

`expiredBehaviors` 现在包含 **4 种显示模式**（单选互斥）+ **1 种通知**（独立勾选）：

| 显示模式 | 说明 |
|---------|------|
| `keep` | 保持显示（100% 满色段常驻） |
| `blink` | 闪烁提醒（该段柔和 alpha 渐变） |
| `hide` | 自动隐藏（从顶栏移出） |
| `celebrate` | 庆祝模式（四边同时闪烁）← **新增** |
| `notify` | 系统通知（勾选框，可叠加在上述任意模式上） |

### 2.3 数据模型

**Go 端 `task.go`：**

```go
// 无需修改，ExpiredBehaviors 已经是 []string
// 只需在验证逻辑中接受 "celebrate"
```

**默认值：**

全局默认 `expiredBehaviors = ["keep"]`（不包含 `celebrate`）。

### 2.4 实现要点

#### `OverlayWindow` 改造

1. 当 `segments[].expired = true` 且 `behaviors` 包含 `celebrate` 时：
   - 如果该任务的位置是"四边模式"，则四边同时闪烁。
   - 如果该任务的位置是单边，则仅该边闪烁（降级处理）。
2. 闪烁动画：使用 `DoubleAnimation`（正弦缓动，淡出/淡入）。
3. 停止条件：用户打开设置（`AcknowledgeBlink`）。

#### `App.xaml.cs` 改造

- 广播中新增 `celebrate` 标记。
- 当收到 `celebrate` 任务时，触发四边闪烁。

### 2.5 配置窗 UI

**任务编辑区「到期提醒」下拉：**

```
[下拉] 使用全局默认 ▼
       ├ 保持显示
       ├ 自动隐藏
       ├ 闪烁提醒
       └ 庆祝模式 ← 新增
```

**全局设置「到期后行为」下拉：** 同步更新。

## 3. 任务级位置覆盖

### 3.1 功能定义

**默认行为：**

- 所有任务使用全局设置的进度条位置（顶/底/左/右/四边）。

**任务级覆盖：**

- 当用户在高级设置中开启"允许任务级位置覆盖"后，任务编辑区显示"展示位置"下拉。
- 用户可为该任务选择不同于全局的位置（仅单边，不支持任务级四边）。
- 渲染时，该任务使用自己的位置，其他任务使用全局位置。

## 4. 任务完成按钮

### 4.1 功能定义

**"完成"按钮：**

- 在任务列表每个任务项后面添加"完成"按钮（图标或文字）。
- 点击后：
  1. 如果该任务是**重复任务**（循环任务），则自动修改 `startAt`/`endAt` 到下一个循环周期，任务继续运行。
  2. 如果该任务是**单次任务**，则标记为"已完成"，不再渲染到桌面，但保留在任务列表中（带"已完成"标记）。
  3. 如果用户想彻底删除，仍使用"删除"按钮。

### 4.2 数据模型

#### Go 端 `task.go`

无需新增字段。`Recurrence` 字段已包含循环信息。

**"已完成"标记（可选）：**

如果要在任务列表中显示"已完成"状态，需新增字段：

```go
type Task struct {
    // ... 现有字段
    CompletedAt *time.Time `json:"completedAt"` // nil = 未完成；非 nil = 完成时间
}
```

### 4.3 实现要点

#### `task.go` 新增 `Complete()` 方法

```go
func (t *Task) Complete(now time.Time) error {
    if t.Recurrence != nil && t.Recurrence.Mode != "" {
        // 重复任务：计算下一个周期
        nextStart := calculateNextOccurrence(t, now)
        if nextStart.IsZero() {
            return errors.New("no next occurrence")
        }
        // 修改 startAt 和 endAt
        duration := t.EndAt.Sub(t.StartAt)
        t.StartAt = nextStart
        t.EndAt = nextStart.Add(duration)
        t.CompletedAt = nil  // 重置完成标记
    } else {
        // 单次任务：标记为已完成
        t.CompletedAt = &now
    }
    return nil
}

// calculateNextOccurrence 计算下一个发生日
func calculateNextOccurrence(t *Task, now time.Time) time.Time {
    if t.Recurrence == nil {
        return time.Time{}
    }
    
    switch t.Recurrence.Mode {
    case "daily":
        // 每天：下一天
        return time.Date(now.Year(), now.Month(), now.Day()+1, 
            t.StartAt.Hour(), t.StartAt.Minute(), 0, 0, t.StartAt.Location())
            
    case "everyN":
        // 间隔 N 天：当前日期 + N
        interval := t.Recurrence.Interval
        if interval < 2 {
            interval = 2  // 最小 2 天
        }
        return time.Date(now.Year(), now.Month(), now.Day()+interval, 
            t.StartAt.Hour(), t.StartAt.Minute(), 0, 0, t.StartAt.Location())
            
    case "weekly":
        // 按星期：找下一个匹配的星期
        // ... 实现略
        
    default:
        return time.Time{}
    }
}
```

#### IPC 命令

```json
{"action":"completeTask","taskId":"uuid"}
```

#### 渲染逻辑

- 当 `CompletedAt != nil` 时，该任务不加入 `BuildLayout()`（不渲染到桌面）。
- 但仍保留在任务列表中，带"已完成"标记。

## 5. 颜色去重优化

### 5.1 功能定义

**现有行为（v0.6）：**

- 保存任务时，校验颜色是否与其他任务重复。
- 如果重复，阻止保存，提示用户修改。

**新行为（v0.7）：**

- 保存任务时，校验颜色是否与其他任务重复。
- 如果重复，**允许保存**，但弹出对话框提示用户：
  ```
  当前颜色 #FF5733 已被以下任务使用：
  - 写文档
  - 买菜
  
  是否继续使用这个颜色？
  
  [是，继续使用]  [否，修改颜色]
  ```
- 用户选择"是"则继续保存；选择"否"则取消保存，返回编辑。

### 5.2 实现要点

#### `ConfigWindow.xaml.cs` 改造

```csharp
private void SaveTask()
{
    // ... 现有校验（名称、时间等）
    
    // 颜色去重校验
    var duplicateTasks = _rows.Where(r => r != _editingRow && 
                                           r.Color.Equals(_editingRow.Color, StringComparison.OrdinalIgnoreCase)).ToList();
    
    if (duplicateTasks.Any())
    {
        var result = MessageBox.Show(
            $"当前颜色 {_editingRow.Color} 已被以下任务使用：\n" +
            string.Join("\n", duplicateTasks.Select(t => "- " + t.Name)) +
            "\n\n是否继续使用这个颜色？",
            "颜色重复提醒",
            MessageBoxButton.YesNo,
            MessageBoxImage.Warning);
            
        if (result == MessageBoxResult.No)
        {
            return;  // 取消保存
        }
    }
    
    // ... 继续保存
}
```

### 5.3 配置窗 UI

无需新增 UI 元素，只需修改保存逻辑。

## 6. 施工顺序建议

建议按以下顺序施工：

1. **颜色去重优化**（最简单，仅修改保存逻辑）

5. **任务级位置覆盖**（最难，需改造多窗口协调）

## 8. 开放问题

1. **四边模式的百分比分配**：当前方案是每边固定 25%。是否应按屏幕周长比例分配？（更复杂，但更精确）
2. **庆祝模式的闪烁频率**：建议 1 秒渐出 + 1 秒渐入，循环直到用户查看。
3. **任务级位置覆盖的优先级**：如果全局是"四边模式"，任务是"顶边"，应该如何渲染？（建议：任务级覆盖优先）
4. **"完成"按钮的图标**：建议使用 ✓ 或 🎯（更有"盼头"的感觉）。
