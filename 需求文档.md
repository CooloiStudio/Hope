#

**🛠️ 项目开发全案技术文档：Hope（盼头）1. 项目简介与核心痛点**

* **项目名称**：Hope（中文名：盼头）
* **目标系统**：Windows 10 / Windows 11 (后续预留 macOS / Linux 扩展空间)
* **核心功能**：一款工作效率提示桌面软件。用户在桌面端设置任务截止时间，系统将在屏幕顶端渲染一根 `1~10px` 高、全宽度的极简进度条。当截止时间临近，进度条将被填满。
* **核心技术痛点**：进度条必须**无条件覆盖独占全屏（Exclusive Fullscreen）的游戏画面**，防止用户因玩游戏而忘记时间；同时要求**0掉帧、0性能损耗、鼠标可完美击穿（不可点击）**。

**2. 核心架构设计 (Headless核心 + Native外壳)**&#x672C;项目拒绝使用传统高能耗、易被游戏反作弊拦截的 DirectX 注入（DX Overlay）技术，采用 **“无头核心 + 原生外壳分离”** 的轻量级双端架构：

1. **Headless 核心 (Golang)**：常驻系统后台（无窗口）。负责任务定时、时间百分比计算、本地配置持久化，通过**Windows命名管道 (Named Pipes)** 向前端 UI 实时广播进度数据。
2. **桌面设置窗体 (C# WPF / WinForms)**：普通的 Win32 窗体。供用户设置任务、截止时间、查看状态，日常可最小化至系统托盘。
3. **游戏覆盖 UI (C# UWP / Xbox Game Bar Widget)**：利用微软官方的游戏栏特权组件。这是唯一能合法穿透独占全屏且不引起掉帧的方案。

text

```
[ 用户设置 UI: C# WPF ] ──控制命令──> [ Headless 核心: Go ]
                                              │
                                        (Windows 命名管道)
                                              ▼
                                   [ 全屏覆盖 UI: Game Bar UWP ] (鼠标击穿, 0损耗置顶)
```

请谨慎使用此类代码。**3. 单仓库代码目录结构 (Monorepo)**&#x8BF7;在本地创建以下目录结构，以便 GitHub Actions 进行统一编译：text

```

├── .github/
│   └── workflows/
│       └── release.yml     # GitHub Actions 自动化编译打包脚本
├── src/
│   ├── headless/           # 1. Golang 后台核心
│   │   ├── main.go
│   │   └── go.mod
│   ├── win-desktop/        # 2. Windows 桌面配置端 (WPF)
│   │   └── WinDesktop.csproj

│   │   ├── Package.appxmanifest
```

请谨慎使用此类代码。**4. 关键技术实现规范A. 进程间通信 (IPC) 突破沙盒限制**

* **机制**：由于 UWP 组件运行在 AppContainer 沙盒中，普通的 Socket 或文件型 Unix 域套接字会被系统拦截。
* **规范**：Go 核心必须建立以 `\\.\pipe\LOCAL\` 开头的命名管道（例如 `\\.\pipe\LOCAL\HopeTimerPipe`），并配置安全描述符，允许 `ALL APPLICATION PACKAGES` 读取。
* **数据格式**：每秒通过管道向 UI 广播一个简单的 JSON，形如：`{"percent": 45.5, "visible": true}`。

**B. Game Bar 进度条组件 (UWP)**

* **页面配置**：全屏拉伸，背景完全透明（`Background="Transparent"`），仅在最顶部保留一根进度条。
* **特性开关**：在组件初始化时，强制开启 **鼠标击穿（Click-Through）**，确保玩家在游戏中划过屏幕顶端时，鼠标操作直接穿透至游戏，不引发误触或切出游戏。csharp

```
widget.ClickThroughEnabled = true; // 允许鼠标击穿
widget.RequestedHeight = 10;       // 极窄高度限制
```

请谨慎使用此类代码。

**C. Go 核心无窗口隐藏运行**

* **规范**：编译时必须添加编译标签 `-ldflags="-s -w -H=windowsgui"`，确保 `headless.exe` 启动时没有黑色的 CMD 命令行控制台弹窗，完全隐形。

**5. CI/CD 自动化分发与全自动打包 (GitHub Actions)**&#x901A;过在云端临时生成自签名证书并调用 Inno Setup，用户最终只会拿到一个干净的 `Hope_Setup.exe` 安装包。**📦 自动化构建流水线逻辑 (`.github/workflows/release.yml`)**

1. **拉取代码**，配置 Go 和 .NET/MSBuild 环境。
2. **编译 Go 核心** 产生 `headless.exe`。
3. **在云端通过 PowerShell 自动创建一张临时的自签名证书**，并将其导出为 `GitHubActionsCert.pfx`。
4. **编译 UWP 组件**，使用上述临时证书为 AppX/MSIX 包进行数字签名。
5. **编译 WPF 桌面端**。
6. **调用 Inno Setup Action** 组合所有编译产物。
7. **全自动生成 Release**，发布唯一的单个安装包。

**🛠️ 安装包静默安装核心设计 (`setup.iss`)**&#x4E3A;了消除用户手动导入证书和旁加载的门槛，Inno Setup 在安装结束前，必须通过管理员权限静默执行 UWP 伴随的 PowerShell 安装脚本：ini

```
[Run]
Filename: "powershell.exe"; \
  Parameters: "-ExecutionPolicy Bypass -File ""{app}\GameBarWidget\Add-AppDevPackage.ps1"" -Force"; \
  Flags: runhidden waituntilterminated; \
  StatusMsg: "正在配置 Xbox Game Bar 屏幕置顶组件，请稍候..."
```
